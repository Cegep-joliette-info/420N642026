# Vidéo individuelle — Démonstration et compréhension du code

**Pondération :** 20% de la note finale
**Durée :** Environ 10 minutes, 15 minutes maximum

---

## 1. Structure de la vidéo (10 minutes)

La vidéo est divisée en deux parties égales :

| Partie | Durée | Description |
|---|---|---|
| **Démonstration** | ~5 min | Présentation des fonctionnalités que vous avez personnellement implémentées |
| **Questions techniques** | ~5 min | Réponses aux 4 questions standardisées ci-dessous, en montrant votre code |

> Utilisez un outil de capture d'écran (OBS, Loom, etc.) pour enregistrer votre écran et votre voix simultanément.

---

## 2. Partie 1 — Démonstration des fonctionnalités

Montrez dans l'interface chaque fonctionnalité que **vous avez développée**. Pour chacune :

- Montrez comment elle fonctionne à l'écran
- Montrez rapidement la gestion d'erreur

---

## 3. Partie 2 — Questions techniques

Répondez aux **3 questions suivantes** en montrant votre propre code à l'écran.

**Q1 — Flux complet**
Choisissez une fonctionnalité et suivez le trajet complet d'une requête : Angular → NestJS → PostgreSQL → retour à Angular. Expliquez chaque étape.

**Q2 — Esprit critique**
Identifiez une limite ou un problème potentiel dans une partie de votre code et expliquez ce que vous changeriez.

**Q3 — Expérience avec l'IA**
Décrivez concrètement comment vous avez travaillé avec un assistant IA durant ce projet : comment vous vérifiiez le code généré, qu'est-ce que cela vous a appris ou au contraire limité, et comment avez-vous apprécié son utilisation.

---

## 4. Consignes de remise

- Format : MP4 ou lien de visionnement (YouTube non répertorié, Microsoft Stream, etc.)
- Qualité audio : voix clairement audible
- Le code doit être lisible à l'écran
- Remise sur Moodle

---

## 5. Grille de correction

| Critère | Excellent | Bon | À améliorer | Problématique |
| ------- | --------- | --- | ----------- | ------------- |
| Présentation des fonctionnalités | Démonstration claire, complète et structurée de toutes ses fonctionnalités<br>5 | Présentation correcte avec quelques imprécisions mineures<br>4 | Présentation incomplète ou peu claire<br>2 | Démonstration très lacunaire ou absente<br>0 |
| Compréhension fonctionnelle (Q1) | Explique clairement le flux complet Angular → NestJS → PostgreSQL avec précision<br>5 | Suit correctement le flux avec quelques imprécisions<br>4 | Flux partiellement expliqué, lacunes sur une couche<br>2 | Peine à suivre le flux ou lit le code sans le comprendre<br>0 |
| Esprit critique (Q2) | Identifie une limite concrète et réaliste, propose une amélioration pertinente<br>5 | Identifie une limite avec quelques imprécisions<br>4 | Limite identifiée de façon superficielle<br>2 | Incapable d'analyser son propre code de façon critique<br>0 |
| Expérience avec l'IA (Q3) | Décrit de façon réfléchie et concrète son flux de travail avec l'IA (prompts, vérification, apprentissage, limites)<br>5 | Décrit son usage de l'IA avec quelques exemples concrets<br>4 | Description vague ou générale sans exemples précis<br>2 | Peu de recul ou refuse de réfléchir à son usage de l'IA<br>0 |
